## 2. 自查结论摘要

Z-Tally 的页面结构、系统返回、基础按钮层级、开关、文本控件、字体字阶、本地化和空历史状态总体符合 Zepp OS 的设计方向。

发布候选版本前需要优先完成以下 3 项必选整改：

1. 应用图标四周增加至少 2px 完全透明安全区。
2. Reset 和历史删除确认改用系统 Modal 组件。
3. 修复 Settings 页面振动说明文字与 Screen timeout 行之间的 2px 几何重叠，并补充清晰间距。

完成以上修改后，还需要对统一字阶、首次教程、中英文长度、历史列表间距和颜色对比度进行一次模拟器及真机回归。

## 4. 定量检查记录

### 4.1 图标透明安全区

| 文件 | 尺寸 | 四角 | 最外侧 1px 最大 Alpha | 最外侧 2px 最大 Alpha | 结论 |
|---|---:|---|---:|---:|---|
| `release/icon-240.png` | 240×240 | 透明 | 255 | 255 | 未形成 2px 全透明安全区 |
| `assets/gt.r/icon.png` | 248×248 | 透明 | 255 | 255 | 未形成 2px 全透明安全区 |
| `assets/gt.s/icon.png` | 248×248 | 透明 | 255 | 255 | 未形成 2px 全透明安全区 |

检查方式：读取 PNG Alpha 通道，统计画布最外侧 1px 和 2px 边框区域内的最大透明度。`255` 表示该边框区域内存在完全不透明像素。

### 4.2 主要颜色对比度

| 组合 | 对比度 |
|---|---:|
| 白色标题 / `color_sys_item_bg` 等效背景 | 13.20:1 |
| 白色按钮文字 / 普通灰色按钮 | 11.73:1 |
| 白色按钮文字 / 系统关键蓝 | 3.91:1 |
| 白色按钮文字 / Save 绿色 | 3.38:1 |
| 白色按钮文字 / 警示红 | 6.09:1 |
| 次要灰色文字 / 黑色背景 | 5.32:1 |

说明：官方自查表要求进行对比度自查，但该表本身没有给出强制数值阈值。本报告仅记录计算结果；最终是否易读仍需结合手表屏幕尺寸、字号和真机亮度验证。

### 4.3 字阶检查

- 常规文字统一定义在 `utils/theme.js`。
- 480×480 圆屏字阶：Caption 24、Subheadline 28、Body 32、Title 36、Title1 40、LargeTitle 48。
- 所有页面常规 `text_size` 已改为引用 `TYPOGRAPHY`，静态检查没有发现直接写入数字的 `text_size`。
- 主计数数字和历史数值按照位数动态缩放，属于功能性大数字例外。
